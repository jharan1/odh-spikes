## Setup Procedure

### 1. Apply ConfigMaps

```bash
# Apply custom model endpoints (includes mistral-embed-provider and granite-embed-provider)
oc apply -f - <<EOF
apiVersion: v1
kind: ConfigMap
metadata:
  name: gen-ai-aa-custom-model-endpoints
  namespace: proj1
data:
  config.yaml: |
$(sed 's/^/    /' config/gen-ai-aa-custom-model-endpoints.yaml)
EOF

# Apply vector stores config
oc apply -f - <<EOF
apiVersion: v1
kind: ConfigMap
metadata:
  name: gen-ai-aa-vector-stores
  namespace: proj1
data:
  config.yaml: |
$(sed 's/^/    /' config/gen-ai-aa-vector-stores.yaml)
EOF
```

### 2. Create the Mistral API Key Secret

```bash
oc create secret generic mistral-api-key \
  --from-literal=api_key=<your-mistral-api-key> \
  -n proj1
```

### 3. Install the LSD from the UI

Delete any existing LSD then install from the Gen AI playground UI, selecting:
- `llama-32-3b-instruct` (namespace model)
- `mistral-embed` (custom endpoint, for Mistral KB)

The BFF generates `llama-stack-config` automatically on install.

### 4. Ingest Test Data into the Mistral Knowledge Base

LlamaStack's file ingestion API uses `remote::passthrough` for the Mistral embedding provider,
which requires the API key in the HTTP request header. Since ingestion runs as an async background
task, the request context (and header) is no longer available when the embedding call is made.

**Workaround: direct pgvector insert with pre-computed Mistral embedding.**

```bash
MISTRAL_KEY=$(oc get secret mistral-api-key -n proj1 -o jsonpath='{.data.api_key}' | base64 -d)
WEATHER_TEXT="Todays weather is sunny with a high of 25C and a low of 15C. There is a 10% chance of rain."

# 1. Call Mistral embeddings API
EMBEDDING=$(curl -s -X POST "https://api.mistral.ai/v1/embeddings" \
  -H "Authorization: Bearer ${MISTRAL_KEY}" \
  -H "Content-Type: application/json" \
  -d "{\"model\": \"mistral-embed\", \"input\": [\"${WEATHER_TEXT}\"]}" \
  | jq -r '.data[0].embedding | "[" + (map(tostring) | join(",")) + "]"')

CHUNK_ID=$(uuidgen | tr '[:upper:]' '[:lower:]')
FILE_ID="file-mistral-direct-insert"
NOW=$(date +%s)
TABLE="vs_vs_bff00004_0000_0000_0000_000000000001"
PG_POD=$(oc get pods -n pgvect -o name | head -1 | sed 's/pod\///')

# 2. Insert directly into pgvector with correct LlamaStack document structure
oc exec -n pgvect $PG_POD -- psql -U vectoruser -d vectordb -c \
"INSERT INTO ${TABLE} (id, document, embedding, content_text, tokenized_content)
VALUES (
  '${CHUNK_ID}',
  '{\"content\": \"${WEATHER_TEXT}\", \"chunk_id\": \"${CHUNK_ID}\", \"metadata\": {\"file_id\": \"${FILE_ID}\", \"chunk_id\": \"${CHUNK_ID}\", \"filename\": \"weather.txt\", \"document_id\": \"${FILE_ID}\", \"token_count\": 39, \"chunk_tokenizer\": \"tiktoken:cl100k_base\", \"metadata_token_count\": 20}, \"chunk_metadata\": {\"source\": null, \"chunk_id\": \"${CHUNK_ID}\", \"document_id\": \"${FILE_ID}\", \"chunk_window\": \"0-39\", \"chunk_tokenizer\": \"tiktoken:cl100k_base\", \"created_timestamp\": ${NOW}, \"updated_timestamp\": ${NOW}, \"content_token_count\": 39, \"metadata_token_count\": 20}, \"embedding_model\": \"mistral-embed\", \"embedding_dimension\": 1024}'::jsonb,
  '${EMBEDDING}'::vector,
  '${WEATHER_TEXT}',
  to_tsvector('english', '${WEATHER_TEXT}')
);"
```

### 5. Verify Ingestion

```bash
PG_POD=$(oc get pods -n pgvect -o name | head -1 | sed 's/pod\///')
oc exec -n pgvect $PG_POD -- psql -U vectoruser -d vectordb \
  -c "SELECT COUNT(*) FROM vs_vs_bff00004_0000_0000_0000_000000000001;"
```

### 6. Verify Search via LlamaStack

```bash
MISTRAL_KEY=$(oc get secret mistral-api-key -n proj1 -o jsonpath='{.data.api_key}' | base64 -d)

oc port-forward svc/lsd-genai-playground-service 18321:8321 -n proj1 &
sleep 3

curl -s -X POST "http://localhost:18321/v1/vector_stores/vs_bff00004-0000-0000-0000-000000000001/search" \
  -H "Content-Type: application/json" \
  -H "X-LlamaStack-Provider-Data: {\"passthrough_api_key\": \"${MISTRAL_KEY}\", \"passthrough_url\": \"https://api.mistral.ai\"}" \
  -d '{"query": "what is the weather today", "max_num_results": 3}' | jq '.data[].content[].text'
```

Expected output:
```
"Todays weather is sunny with a high of 25C and a low of 15C. There is a 10% chance of rain."
```

## Known Issues & Limitations

### `remote::passthrough` Cannot Support Async File Ingestion

LlamaStack's file ingestion (`POST /v1/vector_stores/{id}/files`) spawns an async background
task to chunk and embed content. The `remote::passthrough` provider reads its API key exclusively
from the `X-LlamaStack-Provider-Data` request header at call time. By the time the background
embedding task runs, the original request context (and its headers) is gone, causing:

```
ValueError: Pass API Key for the passthrough endpoint in the header
X-LlamaStack-Provider-Data as { "passthrough_api_key": <your api key>}
```

This affects both the Mistral KB (`vs_bff00004`) and the Weather KB with granite remote embedding
(`vs_bff00003`).

**For RAG queries (responses API):** The BFF's responses handler (`lsd_responses_handler.go`)
injects `passthrough_api_key` and `passthrough_url` into the provider data header for synchronous
requests, so query-time embedding works correctly.

**For ingestion:** Use the direct pgvector insert workaround above (pre-compute the embedding
externally and insert the row with the correct LlamaStack document schema).

### Required pgvector Document Schema

LlamaStack's `EmbeddedChunk` pydantic model requires `chunk_metadata` as a top-level field in
the `document` JSONB column. Missing this field causes a validation error during search result
deserialization, even though the vector similarity search succeeds. The required structure is:

```json
{
  "content": "<text>",
  "chunk_id": "<uuid>",
  "metadata": {
    "file_id": "...",
    "chunk_id": "...",
    "filename": "...",
    "document_id": "...",
    "token_count": 0,
    "chunk_tokenizer": "tiktoken:cl100k_base",
    "metadata_token_count": 0
  },
  "chunk_metadata": {
    "source": null,
    "chunk_id": "...",
    "document_id": "...",
    "chunk_window": "0-N",
    "chunk_tokenizer": "tiktoken:cl100k_base",
    "created_timestamp": 0,
    "updated_timestamp": 0,
    "content_token_count": 0,
    "metadata_token_count": 0
  },
  "embedding_model": "<model-id>",
  "embedding_dimension": 1024
}
```
