# vector-stores-signoff Namespace Config

## ConfigMaps

- `gen-ai-aa-custom-model-endpoints.yaml` — Claude Haiku (Anthropic), Gemini 2.5 Flash (Google), and OpenAI text-embedding-3-small (embedding)
- `gen-ai-aa-vector-stores.yaml` — pgvector and qdrant providers, with one vector store each
- `llama-stack-config.yaml` — full LlamaStack config (generated by operator, manually kept in sync)

## Secrets (not committed — recreate manually)

| Secret name | Key | Source |
|---|---|---|
| `gemini-api-key` | `api_key` | Google Gemini API key |
| `openai-api-key` | `api_key` | OpenAI API key |
| `pgvector-bff-credentials` | `password` | pgvector `vectoruser` password |
| `anthropic-api-key` | `api_key` | Anthropic API key |

Recreate secrets with:
```bash
oc create secret generic gemini-api-key --from-literal=api_key=<key> -n vector-stores-signoff
oc create secret generic openai-api-key --from-literal=api_key=<key> -n vector-stores-signoff
oc create secret generic pgvector-bff-credentials --from-literal=password=<password> -n vector-stores-signoff
oc create secret generic anthropic-api-key --from-literal=api_key=<key> -n vector-stores-signoff
```

## Infrastructure

### pgvector
Pre-existing on the cluster in the `pgvect` namespace (`pgvector2` deployment).
- Host: `pgvector2.pgvect.svc.cluster.local:5432`
- DB: `vectordb`, user: `vectoruser`
- Tables: `vs_vs_signoff_pgvector_001` (pgvector store)

### qdrant
Deployed to the `qdrant` namespace. Config saved in `qdrant-deployment.yaml`.

To redeploy from scratch:
```bash
oc new-project qdrant
oc apply -f qdrant-deployment.yaml
```

The qdrant collection (`vs_signoff-qdrant-001`) is created automatically by the ingest script if it doesn't exist.

## Content

Source content ingested into the vector stores is in the `content/` folder:

- `team-pgvector.txt` — data ingested into the pgvector store (names suffixed with `P`)
- `team-qdrant.txt` — data ingested into the qdrant store (names suffixed with `Q`)
- `example-team-data.txt` — original unsuffixed source data

## Ingestion

Use `ingest_team_data.py` to (re)populate both vector stores. It:
- Calls the OpenAI API to compute embeddings for both texts
- Inserts directly into pgvector via `oc exec` (bypasses LlamaStack's passthrough API key limitation)
- Inserts into qdrant via the qdrant REST API with the correct LlamaStack payload schema
- Creates the qdrant collection automatically if it doesn't exist

```bash
OPENAI_KEY=$(oc get secret openai-api-key -n vector-stores-signoff -o jsonpath='{.data.api_key}' | base64 -d)
PG_POD=$(oc get pods -n pgvect -o name | head -1 | sed 's/pod\///')
python3 ingest_team_data.py "$OPENAI_KEY" "$PG_POD"
```

## Notes

- The pgvector store uses its own isolated table (`vs_vs_signoff_pgvector_001`).
- The qdrant store is entirely separate from any other namespace.
- The `openai-provider` uses `remote::passthrough` — LlamaStack cannot use it for background embedding tasks, hence direct ingestion via the script above.
- The Gemini API key has IP restrictions and may be blocked from the cluster's egress IP. Use Claude Haiku or an unrestricted key for testing if needed.
